DocVision OCR
Multi-Agent AI Document Intelligence Platform — v3.0
DocVision OCR is a production-grade AI system that transforms PDF documents into an interactive knowledge base. Upload any PDF — including scanned and image-based documents — and ask natural language questions. The system extracts, understands, and reasons over your documents using a pipeline of specialized AI agents powered by Groq's LLaMA 3 API.

Features
Core Capabilities

PDF + OCR Processing — Extracts text from both digital and scanned PDFs using PyMuPDF and EasyOCR
Hybrid Retrieval — Combines TF-IDF keyword search with semantic embeddings (Sentence Transformers + FAISS) for highly relevant chunk retrieval
Cross-Encoder Reranking — Re-scores retrieved chunks using a cross-encoder model for maximum relevance
Groq LLM Generation — Answers are generated by LLaMA 3 (70B) via the Groq API — fast, accurate, and grounded
Hallucination Detection — Every answer is validated against retrieved context; low-confidence or ungrounded answers are flagged automatically
Word Report Export — Every Q&A session generates a downloadable .docx report with the question, answer, reasoning notes, and sources

Intelligence Features (New in v3)

Streaming Responses — Answers stream token-by-token in the UI for immediate feedback
Conversation Memory — Maintains context across multi-turn conversations; follow-up questions are understood automatically
Query Type Detection — Automatically classifies queries as summarization, comparison, definition, extraction, steps, example, or factual, and adjusts prompts accordingly
Document Insights — One-click generation of auto-summary, key topics, and difficulty analysis
Smart Notes & Exam Questions — Generates study notes, short-answer questions, long-answer questions, and MCQs from your document
Custom Prompts & Tone Control — Override the system prompt or select response tone (professional, simple, technical, academic)

Transparency & Debugging

RAG Debug Viewer — Exposes every retrieved chunk, TF-IDF score, and rerank score for the last query
Evaluation Dashboard — Displays system metrics: total queries, average response time, chunks indexed, memory turns, LLM backend

Setup
Prerequisites

Python 3.9 or higher
A free Groq API key from https://console.groq.com

Step 1 — Clone or download the project
bashgit clone https://github.com/your-username/docvision-ocr.git
cd docvision-ocr
Step 2 — Create and activate a virtual environment
bashpython -m venv .venv

# Windows
.venv\Scripts\activate

# macOS / Linux
source .venv/bin/activate
Step 3 — Install dependencies
bashpip install -r requirements.txt
Step 4 — Configure your API key
Open .env and replace the placeholder with your actual Groq key:
GROQ_API_KEY=gsk_your_actual_key_here
Step 5 — Run
bashpython app.py
The browser opens automatically at http://localhost:7860.
The FastAPI docs are available at http://localhost:8000/docs.

Usage Guide
Uploading Documents

Open the Gradio UI at http://localhost:7860
Click the upload area and select one or more PDF files (up to 10, max 50 MB each)
Click Process Documents
Wait for indexing to complete — the status box will show the number of pages and chunks processed

Asking Questions

Type your question in the Chat & Q&A tab
Click Get Answer or Stream Answer
The answer appears with confidence score, query type, and source attribution
The Sources panel shows which document chunks were used
The Follow-up Suggestions panel offers related questions to explore

Document Insights
Go to the Document Insights tab and click:

Auto Summary — comprehensive document summary
Key Topics — list of main themes and concepts
Difficulty Analysis — content categorized as Beginner / Intermediate / Advanced

Smart Notes & Exam Questions
Go to the Smart Notes & Questions tab and click:

Generate Smart Notes — structured study notes
Short Answer Questions — 8 factual recall questions
Long Answer Questions — 5 essay-style questions
MCQ Questions — 5 multiple-choice questions with answers
All Question Types — generates all three in one go

Downloading Reports
Go to the Report & Export Center tab:

Download Q&A Report — downloads the Word document from the last answered question
Build & Download Insights Report — generates all insights into a single comprehensive Word document (takes 1-2 minutes)

RAG Debug Viewer
Go to the RAG Debug Viewer tab after asking a question. Click Refresh Debug Data to see:

How many chunks were retrieved and reranked
TF-IDF and rerank scores for each chunk
Text preview of each retrieved chunk
Whether hallucination detection was triggered

Evaluation Dashboard
Go to the Evaluation Dashboard tab and click Refresh Metrics to see:

Total queries run in this session
Average response time
Number of documents and chunks loaded
Full query log with types, confidence scores, and hallucination flags

Deployment on Render
Step 1 — Push to GitHub
bashgit init
git add .
git commit -m "DocVision OCR v3"
git remote add origin https://github.com/your-username/docvision-ocr.git
git push -u origin main
Make sure .env is in .gitignore and is not pushed.
Step 2 — Create a Web Service on Render

Go to https://render.com and sign in
Click New > Web Service
Connect your GitHub repository
Render reads render.yaml and fills in settings automatically

Step 3 — Set environment variables
In the Render dashboard, go to your service and click the Environment tab. Add:
GROQ_API_KEY = gsk_your_actual_key_here
Step 4 — Deploy
Click Deploy. Render runs:
pip install -r requirements.txt
python start_render.py
After deployment, your app is live at:
https://docvision-ocr.onrender.com/gradio    Gradio UI
https://docvision-ocr.onrender.com/docs      API documentation
https://docvision-ocr.onrender.com/health    Health check
Render notes:

The free/starter tier sleeps after 15 minutes of inactivity. Upgrade to the Standard plan ($7/month) for always-on.
EasyOCR downloads approximately 200 MB of models on first run. This is normal and cached automatically afterward.
start_render.py mounts Gradio inside FastAPI using gr.mount_gradio_app() so both services run on Render's single exposed port.


Limitations

All processed documents are held in memory for the current server session. Restarting the server clears all state.
For persistent storage across restarts, integrate PostgreSQL with SQLAlchemy to store document metadata, chunks, and history.
For scalable vector search beyond a single server, replace the FAISS index in core.py with Pinecone.
The free Groq API tier has rate limits. If you hit them frequently, the system automatically retries with the fallback model (llama-3.1-8b-instant).
